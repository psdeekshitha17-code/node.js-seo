File System in Node.js

Beginner Guide to Node.js fundamentals

The File System (fs) module in Node.js allows developers to interact with the file system of the computer. It provides methods to read, write, update, delete, and manage files. This module is one of the most important core modules in Node.js and is widely used in backend development.

Using the File System module, applications can store data, read configuration files, and manage user uploads. Node.js provides both synchronous and asynchronous methods to perform file operations.

---

Importing File System Module

To use the file system module, we need to import it using require(): 

const fs = require('fs');
---

Reading Files

Node.js allows reading files asynchronously, which is more efficient. 

const fs = require('fs');

fs.readFile('file.txt', 'utf8', (err, data) => {
    if (err) throw err;
    console.log(data);
});

This method reads the file without blocking other operations.

---

Writing Files

We can create or overwrite files using writeFile(): 

fs.writeFile('file.txt', 'Hello Node.js', (err) => {
    if (err) throw err;
    console.log("File created");
});
---

Appending Data

To add data without overwriting: 

fs.appendFile('file.txt', '\nNew Data', (err) => {
    if (err) throw err;
});
---

Deleting Files

fs.unlink('file.txt', (err) => {
    if (err) throw err;
    console.log("File deleted");
});
---

Synchronous vs Asynchronous

Node.js provides two types of methods:

  • Synchronous: Blocks execution until task completes
  • Asynchronous: Executes without blocking
fs.readFileSync('file.txt', 'utf8');
---

Advantages of File System Module

  • Easy file handling
  • Supports async operations
  • Efficient for large applications
  • Works on all platforms
---

Real-world Applications

  • Reading configuration files
  • Handling uploads
  • Logging system data
  • Storing user information
---

Conclusion

The File System module is an essential part of Node.js. It allows developers to interact with files and directories efficiently. Understanding file handling is important for building real-world applications such as servers, APIs, and data-driven systems.
